§01 — The problem

Your copy lives in eleven places. They disagree.

The design file says “Sign up”. The spreadsheet says “Create account”. The code shipped “Get started”. Nobody decided this — it just happened, one paste at a time. Your users noticed before you did.

§02 — What it does

Four things. Done properly.

[A]

One panel, every string

Every text layer in the file — or just your selection — listed in one place, grouped by frame. Edit in the panel and the canvas updates as you type. Fonts stay intact.

[B]

A status for every word

None, WIP, Review, Final. Each string carries its state — color-coded, searchable, filterable. You always know which words are done and which are still pretending.

[C]

Reusable copy components

Write a string once and make it a component. Link every other place it appears. Change it once; it changes everywhere. That is the whole trick.

[D]

Clean developer handoff

A read-only view of every string and its status, and a one-click JSON export. Engineers get the words as written — not as remembered.

§03 — Get started

Installed in about a minute.

  1. 1

    Open Figma on the desktop.

  2. 2

    Go to Plugins → Development → Import plugin from manifest.

  3. 3

    Select Idem’s manifest.json.

  4. 4

    Run Plugins → Development → Idem.

Then: open a file → every string appears → edit, set status, componentize → export for the people who ship it.

§04 — The idea

Say it once.

Idem is Latin for “the same as already stated” — the small mark scholars used to say a thing, once written, need not be written differently again. That is the entire promise of this plugin. Decide on the words once, and everywhere they appear, they stay decided.

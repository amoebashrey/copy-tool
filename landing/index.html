// ०१ · the problem

One button. Three labels.

Copy scatters. The button says “Sign up” in the mockup, “Create account” in the doc, and “Get started” in the build. Nobody decided that. It happened one paste at a time, and users spot it before the team does.

It compounds. A stale string ships because the doc moved and nobody told the file. A typo survives review because the source of truth was a screenshot in a Slack thread. Fixing any of it means hunting through frames by hand.

Chitra exists to do the hunting.

// the name

Named for Chitragupta.

In Hindu myth, Chitragupta keeps the complete record of every word and deed. Nothing gets past the ledger. Chitra borrows the name and the job, at a smaller scale: the text layers in a Figma file. She remembers what each one is supposed to say, and when one changes on its own, she says so.

A ledger, kept properly.

// ०२ · what it does

In the plugin right now.

अ

Every string, one panel

Every text layer in the file, in one list, grouped by frame. Edit a line here; the canvas updates. Fonts stay put.

A status on every line

Mark a string None → WIP → Review → Final. Open the file a week later and you can tell which words are settled and which are still moving.

“This one is Final. You may ship it.”

इ

Copy tokens

Turn a string into a token and reuse it across frames. Edit the token once; every linked layer takes the change. The pricing page stops disagreeing with the checkout.

◈ cta_primary Create account
  • Home/HeroCreate account
  • Pricing/CardCreate account
  • Checkout_ModalCreate account

“Change it once. I will inform the other three hundred.”

ई

Handoff without retyping

Engineers get a read-only view and a one-click export to copy.json — token names, text, status. Nobody retypes from a screenshot.

Inscribing copy directly into your codebase…

{
  "checkout_modal": {
    "cta_primary": {
      "text": "Create account",
      "status": "final"
    },
    "cta_secondary": {
      "text": "Maybe later",
      "status": "review"
    }
  }
}

// ०४ · install

Runs as a development plugin.

No marketplace listing yet. Four steps:

  1. १

    Open the Figma desktop app.

    Development plugins don’t run in the browser.

  2. २

    Go to Plugins → Development → Import plugin from manifest…

    You only do this once.

  3. ३

    Pick Chitra’s manifest.json

    It ships in the plugin folder.

  4. ४

    Run Plugins → Development → Chitra

    The panel opens on the right. That’s it.

Select a frame → Chitra lists every string → assign tokens, set statuses → export copy.json for the build.
